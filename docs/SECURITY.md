# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 2.4.x   | :white_check_mark: |
| < 2.4   | :x:                |

## Reporting a Vulnerability

We take security seriously. If you discover a security vulnerability, please follow these steps:

1. **Do not disclose publicly** - Please do not discuss the vulnerability in public channels
2. **Email security report** - Send details to: security@cyberzilla.example.com
3. **Include details**:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if any)

## What to Expect

- **Acknowledgement**: You will receive an acknowledgement within 48 hours
- **Investigation**: We will investigate and validate the report
- **Fix**: If valid, we will work on a fix
- **Disclosure**: We will coordinate disclosure with you
- **Credit**: You will be credited (if desired)

## Security Best Practices for Users

1. **Keep updated**: Always use the latest version
2. **Secure environment**: Run in a secure, isolated environment
3. **Limit permissions**: Run with minimal required permissions
4. **Monitor logs**: Regularly check application logs
5. **Use responsibly**: Only scan systems you have permission to test

## Security Features

- Input validation and sanitization
- Rate limiting for API calls
- Secure database connections
- No storage of sensitive credentials
- Regular security updates
